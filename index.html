django-deployer

Generic Deployment tool for Django that will support PaaS providers such as Heroku, Dotcloud, OpenShift, Amazon Elastic Beanstalk, Google App Engine, Microsoft Azure, Stackato and Gondor

Download this project as a .zip file Download this project as a tar.gz file

The goal of django-deployer is to minimize the effort to deploy a Django app to any of the popular PaaS providers. By asking a series of questions about the Django app, we can generate a generic deploy.yml file that can be consumed by any of the PaaS providers. Each PaaS provider can translate these requirements into a configuration that will work to deploy to their PaaS.

PaaS providers

Here are instructions from each PaaS provider for how to deploy Django to their PaaS:

  • Heroku
  • DotCloud
  • Gondor.io
  • OpenShift
  • Stackato
  • Amazon Elastic Beanstalk
  • Google App Engine
  • Microsoft Azure

Usage

Install django-deployer in your virtualenv, and run the deployer-init command to copy the fabfile into your root folder. Then run the fab setup command to tell django-deployer about your project.

$ cd <root_of_your_project>

$ pip install django-deployer (this won't work until a release has been made to PyPi, so in the meantime, use the following)
$ pip install https://github.com/natea/django-deployer/archive/master.zip
$ deployer-init
$ fab setup

Development

If you want to contribute to django-deployer, here is how you can check it out and install it.

$ git clone git://github.com/natea/django-deployer.git
$ cd django-deployer
$ python setup.py develop
